Project Management Process
Concepto Académico
Concepto: El Project Management Process es la aplicación sistemática de conocimientos, habilidades, herramientas y técnicas (PMI) a través de un sistema integrado de procesos interrelacionados (Kerzner, ISO) para planificar, organizar, dirigir y controlar las actividades de un proyecto (Meredith & Mantel). Este proceso se estructura en cinco grupos: iniciación, planificación, ejecución, monitoreo y control, y cierre, donde cada fase incluye tareas con códigos identificadores que facilitan la gestión integral, asegurando el cumplimiento de requisitos de calidad, tiempo y recursos dentro de los objetivos establecidos.

Definiciones de las Fuentes de Referencia

PMI (2021): Define el Project Management Process como "la aplicación de conocimientos, habilidades, herramientas y técnicas a las actividades del proyecto para cumplir con los requisitos del proyecto" (p. 4). El PMI estructura este proceso en cinco grupos: iniciación, planificación, ejecución, monitoreo y control, y cierre.

Kerzner (2022): Conceptualiza el Project Management Process como "un sistema integrado de procesos que se utilizan para planificar, organizar, dirigir y controlar las actividades de un proyecto" (p. 23). Enfatiza la importancia de la planificación sistemática y el control continuo.

ISO 21500 (2021): Establece que el Project Management Process es "un conjunto de procesos interrelacionados que se utilizan para dirigir y controlar un proyecto" (p. 8). La norma internacional destaca la importancia de la gestión de la calidad, el tiempo y los recursos.

Meredith & Mantel (2019): Definen el proceso como "la aplicación de conocimientos, habilidades, herramientas y técnicas para satisfacer los requisitos del proyecto" (p. 15). Su enfoque se centra en la gestión de equipos y la resolución de problemas.

Síntesis Personal: Basándome en estas definiciones autorizadas, he desarrollado un concepto integral que integra los elementos comunes de planificación sistemática, control de procesos, gestión de recursos y cumplimiento de objetivos, adaptado específicamente para el contexto de gestión de proyectos con códigos de tareas identificables.
